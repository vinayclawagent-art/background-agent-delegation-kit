# Delegation Trial Operator Handoff Checklist

Status: operator-ready; evidence pending. This checklist sequences the existing delegation packet, evidence index, rubric, promotion card, and debrief so the next operator does not skip proof capture.

## Before the trial

- [ ] Open [[VinClawLabs Delegation Card Trial Packet]] and name the real task, business owner, inputs, tools, output, review gate, and kill switch.
- [ ] Open [[Delegation Trial Evidence Index]] and create empty rows for source prompt, run log, output artifact, human review, and follow-up decision.
- [ ] Open [[Delegation Evidence Quality Rubric]] and note the minimum evidence bar before the agent starts.
- [ ] Confirm no README, skill, or public-proof claim will be strengthened until evidence is linked.

## During the trial

- [ ] Paste the delegation-card export or prompt into the trial packet.
- [ ] Save links to raw run logs, generated artifact, reviewer notes, and any rejected output in the evidence index.
- [ ] Record where the background agent needed human intervention.
- [ ] Preserve failure evidence; do not rewrite the trial as success-only proof.

## After the trial

- [ ] Complete [[Post-Trial Debrief Template]] with what changed, what failed, and what claims must stay narrow.
- [ ] Grade proof quality in [[Delegation Evidence Quality Rubric]].
- [ ] Fill [[Delegation Promotion Decision Card]] and choose promote / pilot-only / iterate / hold.
- [ ] Patch README/prototype/skill wording only if the promotion card cites evidence for the exact claim.
